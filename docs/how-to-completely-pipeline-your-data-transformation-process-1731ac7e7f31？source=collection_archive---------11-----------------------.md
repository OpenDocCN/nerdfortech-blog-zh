# 什么是管道？

管道基本上是指将**业务问题**转换为**业务答案的过程。但是要将一个业务问题转换成业务答案，数据工程师应该能够执行某些引入数据问题和数据答案的任务。**

所以，最后整个过程可以总结成这样:

> **商业问题- >数据问题- >数据回答- >商业回答**

基本上，上面的陈述意味着你的整个数据科学项目。这是否意味着您必须为项目中的每一条语句创建一个管道？号码

# 如何使用管道？

**一个数据科学项目可以分为以下几个部分:**

*   获取数据
*   数据可视化
*   数据预处理
*   型号选择
*   模型调整
*   解决方案演示
*   维护模型

这是相当多的，如果你在知道代码会变得非常难以管理之前就已经从事过一个项目。

因此，如果您可以为需要大量代码调整的项目部分创建一个管道，那么项目将变得更容易管理。
