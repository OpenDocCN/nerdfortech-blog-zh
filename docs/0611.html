10分钟内建立一个自定义域名的Next.js网站

原文：https://medium.com/nerd-for-tech/build-a-next-js-website-with-custom-domain-in-10-minutes-71fcb664a845?source=collection_archive---------0-----------------------#2021-01-27

在本教程中，我们将使用create-next-app、Github、Netlify和namescheap部署一个带有自定义域名的Next.js静态网站。

Next.js是一个使用React的“生产就绪”库，具有额外的好处，如服务器端渲染和开箱即用。欲了解更多信息，请查看nextjs.org的。

当然，域名注册和网站托管还有其他选择，但Namescheap提供了我所发现的最便宜的域名，易于使用，还免费提供whoIs隐私保护(因此当你注册一个新域名时，你不会收到垃圾邮件和电话)。Vercel是一个非常简单的托管平台。

注意:此方法仅适用于使用Next.js生成的静态站点。动态站点将需要node.js托管的后端。

第1步，共5步:创建您的应用程序

首先，使用create-next-app:创建一个名为‘my-awesome-app’的新Next.js应用程序(当然，您可以随意更改应用程序的名称)

在终端中，键入:

npx create-next-app my-awesome-app
cd my-awesome-app

重要提示:在package.json文件中，将"build"命令改为"next build && next export"

第2步，共5步:将你的应用程序推送到GitHub

接下来，为项目创建一个新的GitHub repo，然后将您的项目上传到其中。

在GitHub中，点击左上角的“新建”按钮:

键入应用程序的名称，然后单击屏幕底部的“创建存储库”按钮:

从命令行复制代码以推送您的现有存储库:

将代码粘贴到终端以上传您的代码。

第3步，共5步:为托管设置Netlify

接下来，登录 Netlify (如果你还没有的话，创建一个账户)。

点击“来自Git的新网站”

在下一个屏幕上，选择“GitHub”

在下一个屏幕上，向下滚动找到您的GitHub repo for the app:

然后，在下一个屏幕上单击“部署站点”

对于“构建命令”类型yarn build，对于“发布目录”类型out/

接下来，点击“域设置”

接下来，点击“添加自定义域”

键入您的域，然后单击验证

然后单击“是，添加域”

返回到域设置屏幕，复制。app链接。您将在注册自定义域后使用它。

第4步，共5步:在Namescheap注册您的自定义域名

在域名页面上注册域名

购买完成后，进入您的仪表板，找到您想要转让的域名，然后单击“管理”按钮

接下来，点击“高级DNS”

点击“添加新记录”

向下滚动到CNAME，然后单击

对于“主机”，键入“www ”,对于“值”,粘贴您的。来自Netlify的应用链接。点击绿色复选标记，您就可以开始了。

第5步，共5步:将您的域添加到Netlify

您的自定义域名现在应该显示在Netlify上:

前往您的新网站，并开始定制:)。当你推你的GitHub repo你的网站会自动更新。
