创建多级管道并使用构建管道插件进行查看

项目链接= >【https://github.com/AbhishekYuvi/SampleWebApp T4】

第一步

在你的系统上安装Git，Jenkins，Tomcat，Ant。

第二步

  • 在c盘创建“gitworkspace”文件夹
  • 在gitworkspace中，从github克隆存储库
  • 使用“dir”进行检查

第三步

创建一个提取代码的作业

  • 管理Jenkins->管理插件-> github->无需重启安装。
  • 新工作-> github拉->自由式项目->确定
  • Github pull->配置->常规->高级->使用自定义工作区

${JENKINS_HOME}/workspace/samplewebapp

  • SCM-> git->复制github repo url并粘贴

第四步

持续构建->构建、代码评审和发布结果

  • 管理Jenkins->管理插件-> Ant和警告下一代插件。
  • 新项目->构建和代码审查->自由格式项目->确定
  • 常规->高级->使用自定义工作空间

${JENKINS_HOME}/workspace/samplewebapp

  • SCM-> git->从github粘贴Samplewebapp的URL
  • 构建->调用ant
  • 后期生成操作->发布结果分析-> checkstyle_error.xml->保存

第五步

持续测试——运行测试并发布JUNIT测试报告

  • 安装Junit实时测试报告程序插件
  • 新项目->单元测试->自由式项目->确定
  • 常规->高级->使用自定义工作空间

${JENKINS_HOME}/workspace/samplewebapp

  • SCM-> git->从github粘贴URL
  • 构建->调用Ant
  • 后期构建操作->发布Junit测试结果->测试计算器->保存

第六步

设置新的部署服务器

转到Tomcat/conf/server.xml(在您的系统上)

连接器端口“8090”

  • 转到Tomcat/conf/tomcat-users.xml

<user . name = " admin " password = " admin " roles = " manager-GUI，manager-script，admin"/ >

第七步

持续部署将代码部署到生产环境中

  • 安装“部署到容器”插件
  • 新建项目->部署->自由格式项目->确定
  • 常规->高级->使用自定义工作空间

${JENKINS_HOME}/workspace/samplewebapp

  • SCM-> git->从github粘贴URL
  • 构建->调用Ant，目标- WAR
  • 后期生成操作->将war/ear部署到容器(war/ear文件)

添加容器-> Tomcat 9.0

网址:-https://localhost:8090->保存

停止并启动Tomcat 9。exe文件

第八步

  • 安装“构建管道”插件
  • 转到Githubpull->配置->后期生成操作-生成其他项目-> BuildandCodeReview->保存
  • 转到buildanddereview->配置->后期生成操作->生成其他项目->单元测试->保存
  • 转到单元测试->配置->后期生成操作-生成其他项目->部署->保存
  • 点击“+”->完成管道->构建管道视图->确定
  • 选择初始作业-> Githubpull->确定
  • 转到Githubpull->配置->轮询SCM-> * * * * *
  • 您将被重定向到您的管道视图。

绿色:执行完毕，蓝色:尚未开始，黄色:正在执行，红色:失败
